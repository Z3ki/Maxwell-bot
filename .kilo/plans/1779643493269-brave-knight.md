# XML/Tag-Based Tool Calling Implementation Plan

We will transition Maxwell's tool-calling format from JSON to a highly robust XML/tag-based format. This eliminates JSON parsing failures (nested quote-escaping, newlines, syntax errors) and improves overall execution accuracy.

---

## 1. Scope of Changes

We will modify three key areas:
1. **Tool-Call Parsing (`bot.py`):**
   - Completely replace JSON parsing in `collect_tool_calls` with robust XML/tag parsing.
   - Support self-closing tags (`<tool:react emoji="catjam" />`).
   - Support tags with sub-elements (`<tool:send_file><filename>script.py</filename><content>...</content></tool:send_file>`).
   - Support default fallback parameters for raw body text (e.g. `<tool:send_message>hello</tool:send_message>` automatically maps to `content="hello"`).
   - Support both namespace `<tool:react>` and plain `<react>` tags.
   - Rewrite `strip_tool_payload_leaks` to robustly strip tag blocks.
2. **Prompts (`bot.py`):**
   - Update `_tool_system_prompt` to instruct the model on the new XML/tag format, valid examples, and ordering rules.
   - Update the tool-result follow-up system prompt to use tag examples instead of JSON examples.
3. **Tests (`tests/`):**
   - Adapt `tests/test_tool_calls.py` to assert XML/tag parsing, self-closing tags, pretty sub-tags, raw body fallbacks, and leak stripping.
   - Adapt `tests/test_tts_silent.py` to mock and call tools using XML/tag syntax.

## 3. Test Cases to Transition

We will rewrite all test cases in `tests/test_tool_calls.py` to cover:
- Plain tags with attributes (`<tool:react emoji="catjam" />`).
- Tags with sub-tags (`<tool:web_search><query>openrouter</query></tool:web_search>`).
- Sub-element tag parsing with multiline and raw character inputs.
- Default body fallbacks.
- State-based/leak stripping.

We will run `pytest tests/test_tool_calls.py tests/test_tts_silent.py` to verify full correctness of the parser and tool behavior.
