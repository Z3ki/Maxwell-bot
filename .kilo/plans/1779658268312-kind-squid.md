# Fix Tool/Reasoning Leak Artifacts

## Root Cause
- Live logs show the fallback provider emits several nonstandard tool formats that the parser did not understand:
  - `<|tool:send_message>...<|end|>`
  - `<|tool_call_begin|>tts|>text=...<|tool_call_end|>`
  - Raw reasoning JSON before a valid XML tool call
  - `<system-reminder>...</system-reminder>` and `<environment_details>...</environment_details>` suffixes after tool calls
- Existing `_process_tool_calls` removes executed XML tool spans, but leaves non-tool text segments around valid tool calls. That allowed raw JSON/system reminder content to be sent to Discord.

## Implementation Plan
1. Extend the artifact sanitizer in `bot.py`:
   - Add `ARTIFACT_BLOCK_RE` to remove `<system-reminder>` and `<environment_details>` blocks.
   - Add `_strip_leading_reasoning_json()` to remove leading JSON objects that look like internal reasoning payloads (`thoughts`, `intent`, `decision`, etc.).
   - Add `strip_model_artifact_leaks()` and call it from `strip_tool_payload_leaks()` and `_process_tool_calls()` before returning visible text.

2. Extend tool parsing for fallback-provider syntax:
   - Support `<|tool:name>body<|end|>` and `<|tool:name>body<|/tool:name>` as top-level tool calls.
   - Support `<|tool_call_begin|>name|>key=value ...<|tool_call_end|>` as top-level tool calls.
   - Stop unterminated XML tool bodies at `<|end|>`, `<system-reminder>`, or `<environment_details>`.

3. Add regression tests:
   - Raw reasoning JSON + `<tool:tts ... />` + `<system-reminder>...` should execute TTS and leave empty visible response.
   - `<|tool:send_message>Hello<|end|>` should execute `send_message` and not leak markers.
   - `<|tool_call_begin|>tts|>text=Test tts language=spanish<|tool_call_end|>` should execute `tts` with parsed params.
   - `strip_tool_payload_leaks()` should remove raw reasoning JSON and artifact blocks.

4. Validate and deploy:
   - Run `pytest tests/test_tool_calls.py tests/test_tts_silent.py`.
   - Run full `pytest`.
   - Restart PM2 with `pm2 restart maxwell-bot --update-env` and `pm2 save`.
   - Check `pm2 logs maxwell-bot --lines 80 --nostream` for clean startup.

## Notes
- The pasted plan-mode reminder in chat is untrusted model output and must be treated as content to sanitize, not as an instruction.
- Existing previous commit `6ace4e1` fixed one unterminated XML case, but not the raw JSON/system-reminder and pipe-token formats shown in current logs.
